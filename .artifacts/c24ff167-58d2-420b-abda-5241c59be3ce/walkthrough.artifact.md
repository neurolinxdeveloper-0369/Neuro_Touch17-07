# Walkthrough - Development OTP Display

I have implemented a development-friendly way to log in without a real SMS gateway. The OTP is now generated by the backend and sent back directly to the mobile app for display.

## Changes Made

### Backend
- **[auth_controller.go](file:///B:/IoT_Neuro Touch/IoT_Neuro Touch/backend/controllers/auth_controller.go)**: Updated `SendOTP` to include the `otp` field in the JSON response.

### Mobile App
- **[auth.repository.dart](file:///B:/IoT_Neuro Touch/IoT_Neuro Touch/mobile/lib/data/repositories/auth.repository.dart)**: Modified `sendOtp` to return the OTP string from the API response.
- **[auth.controller.dart](file:///B:/IoT_Neuro Touch/IoT_Neuro Touch/mobile/lib/controllers/auth.controller.dart)**: Propagated the OTP string back to the UI.
- **[login_screen.dart](file:///B:/IoT_Neuro Touch/IoT_Neuro Touch/mobile/lib/presentation/auth/login_screen.dart)**: Added a SnackBar that displays the received OTP.

## How to Test
1. Run the backend server.
2. Open the mobile app and go to the Login screen.
3. Enter your phone number and tap **"Send OTP Verification"**.
4. A message will appear at the bottom of the screen saying: **"Debug: Verification code is XXXXXX"**.
5. You will be automatically navigated to the verification screen where you can enter that code.

> [!WARNING]
> Remember to remove the `otp` field from the backend response and the SnackBar from the UI before deploying to production!
